# 作業時間記録(技能五輪 練習ログ)

工程別(原寸図・削り・墨付け・加工・組立)の練習タイムを記録し、選手ごとの苦手工程や
採点誤差の傾向を把握するためのWebアプリです。

- 管理者(指導者)がタイムと気づきメモを入力
- 選手は自分のページを見て、記録の推移や指導者からのメモを確認できます
- 年度ごとにデータを残せます(選手は毎年入れ替わる想定)

## 技術構成

- Next.js 15(App Router)/ TypeScript / Tailwind CSS v4
- Firebase(Firestore + Authentication)
- Vercel(ホスティング・自動デプロイ)

## セットアップ手順

### 1. Firebaseプロジェクトを作成

1. https://console.firebase.google.com/ で新規プロジェクトを作成
2. 「Firestore Database」を作成(本番モードでOK。ルールは後述の`firestore.rules`を貼り付け)
3. 「Authentication」→「Sign-in method」で「メール/パスワード」を有効化
4. 「Authentication」→「Users」で指導者用アカウントを1つ作成(メールアドレス+パスワード)
   → これが管理画面(`/admin`)のログイン情報になります
5. 「プロジェクトの設定」→「マイアプリ」でWebアプリを追加し、表示された設定値を控える

### 2. Firestoreルールを設定

Firebaseコンソールの Firestore →「ルール」タブに `firestore.rules` の内容を貼り付けて公開してください。

> 現在の設定は「読み取りは誰でも可、書き込みは管理者ログイン済みのみ」という方針です。
> 選手の記録ページ(`/view/[id]`)にリンクを知っている人なら誰でもアクセスできる想定のため、
> 機微な情報は入力しないでください。より厳密にアクセス制限したい場合は選手ごとのログインを
> 追加する拡張が必要です。

### 3. 環境変数を設定

`.env.local.example` を `.env.local` にコピーし、Firebaseの設定値を入力してください
(Vercelにデプロイする場合は、Vercelのプロジェクト設定 → Environment Variables に同じ値を登録します)。

### 4. GitHubへアップロード / Vercelでデプロイ

いつも通りGitHubの新規リポジトリにこのフォルダの中身をアップロードし、Vercelでインポートして
デプロイしてください。デプロイ時に環境変数(`NEXT_PUBLIC_FIREBASE_*`)の設定を忘れないようにしてください。

### 5. 初期データ投入

1. デプロイ後、`/admin` にアクセスして手順1で作成したアカウントでログイン
2. 「新しい年度を作成」で年度(例:2026年度)を作成
3. 選手名を追加していく
4. 各選手のページから記録・メモを入力

## 使い方の概要

- トップページ(`/`):選手一覧 → 自分の名前をタップすると記録閲覧ページへ
- `/admin`:指導者用。年度・選手管理、記録入力、メモ入力
- `/view/[id]?year=...`:選手用の閲覧専用ページ

## ローカル開発

```bash
npm install
npm run dev
```
